### [WORK] 트랙 콘텐츠/수강 관리 기능
운영툴에서 콘텐츠를 조회할 때 선택 옵션 정보도 같이 노출되게 하고, 이 정보로 상세 조회와 회수까지 가능하도록 처리(CORE-816). 트랙 상품 상세에서 수강 인원을 관리할 수 있게 조회/필터 기능을 설계했고, 취소 처리는 별도 작업으로 분리(CORE-804). 외부 국비지원 경로로 신청한 수강 건은 대기/수강 확정/취소 상태를 운영자가 관리할 수 있어야 해서 상태 전환 정책을 정리(CORE-807). 주문/결제 후 트랙 입과 시 선택한 옵션 정보를 콘텐츠 접근 데이터에 저장하고, 중복 신청 검증과 취소 시 해당 옵션 건만 회수되도록 처리. 기존 예약 처리 로직은 제거함(CORE-814).

### [WORK] 결제 취소 처리 개선
관리자가 결제를 취소할 때 콘텐츠 회수, 쿠폰 정리, 마일리지 회수 등이 웹훅 처리에 의존하고 있었는데, 이걸 즉시 처리되게 로직 수정. 웹훅 중복 방지 로직 때문에 실제 처리가 누락되던 문제였음(CORE-822).

### [WORK] 모니터링/집계 기능 검토
서버 애플리케이션의 실시간 상태와 성능 지표를 볼 수 있는 모니터링 도구 도입을 테스트 삼아 검토 중(CORE-732, 진행 중). 도서 베스트셀러 노출 방식도 수기 집계에서 주문/리뷰/좋아요 등 지표 기반 자동 집계로 바꾸는 방안을 검토(CORE-676, 진행 중).

### [WORK] 콘텐츠 접근 구조 변경 마무리
선택 옵션 단위 유일성 제약으로 바꾸는 작업(CORE-813)을 마무리. 트랙 주문/결제/입과 전체 플로우에서 중복 주문 방지와 예약 구매 시 마이콘텐츠 조회가 정상 동작하는지 테스트하고, 운영툴 결제 취소 시 콘텐츠 입과 취소가 안 되는 버그도 확인.

### [WORK] 레거시 이벤트 페이지 개선
레거시 사이트의 인클루드 방식 이벤트 페이지는 파일을 배포 저장소에 직접 넣어야만 반영됐는데, 파일 업로드만으로 반영되게 스토리지 연동 방식으로 변경. 기존 자료도 옮기고 상세 페이지 경로 처리도 수정(CORE-840).

### [WORK] 상품 동기화 배치 설계
운영 환경에 새로 등록된 상품 데이터를 스테이지 환경에도 주기적으로 동기화하는 배치를 설계. 대상 환경 구분은 환경변수로 관리하고, 주기 실행 말고 수동 트리거로도 돌릴 수 있게 검토(CORE-765, 진행 중).

### [WORK] 리뷰/입과/쿠폰 관련 소소한 개선
리뷰 응답에서 구매자 여부 처리를 특정 대상만이 아니라 전체 대상으로 처리하도록 수정(CORE-834 후속). 입과 기능에 필터링 옵션을 추가하고, 운영툴 쿠폰 상세페이지에서 쿠폰 메모가 보이도록 처리. 카카오톡 서버 에러 처리는 아직 확인 중.

### [WORK] 버디 프로젝트 착수
버디 신규 기능을 위한 테이블/엔티티 세팅 시작. 버디 자체 엔티티는 완료했고, 회원-버디 매핑은 다음 작업으로 예정(CORE-810, CORE-811로 이어짐).

### [WORK] 결제 상태별 처리 정책 설계
결제 완료 API는 주문 번호와 결제 성공/실패 여부를 받아 주문 내역 기준 마이 콘텐츠 입과 처리, 주문 상태 변경, 마일리지 적립을 수행하도록 설계. 결제 완료(PAID)/결제 대기(PENDING)/대기(READY-CREATE)/결제 실패(FAILED)/취소(CANCELLED)/부분 취소/가상계좌 발급 상태별 policy를 정의하고, 그중 결제 대기·대기·결제 실패 상태의 주문/결제 상태 변경과 pgTxId 저장 로직을 우선 완료. 쿠폰·적립금 사용 확정 처리는 쿠폰 시스템 개편(CORE-4)에서 다루는 사용이력·적용조건 관리와 맞닿아 있는 부분.
결제 완료 상태의 마이콘텐츠 입과, 적립금 적립, 주문 완료 이메일 발송과 취소 상태의 콘텐츠/적립금 회수·복원 로직은 아직 미완료.

### [WORK] 결제/쿠폰 관련 이슈 메모
- 결제 후 취소/환불은 종이책만 가능하며, 종이책은 배송 준비 전 단계에서만 구매 취소 가능
- 주문 생성 시 member_coupon 상태 변경 처리가 필요하나, 현재 주문에 어떤 쿠폰을 사용했는지 정보가 없는 상태
- 복합 결제(결제가 2건)인 경우 하나라도 실패하면 최종 주문은 실패로 처리해야 함

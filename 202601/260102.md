### [WORK] 결제 상태별 주문 상태 처리 조건표 정리
1개의 주문에 결제가 2건까지 있을 수 있다는 가정 하에, 결제 상태(결제완료/취소/실패/대기/준비/가상계좌 발급)에 따른 주문 상태 확정 규칙을 정리함.
- 모든 결제가 완료 → 주문 완료(종료)
- 모든 결제가 취소 → 주문 취소(종료)
- 모든 결제가 실패 → 주문 실패(종료)
- 대기 포함 → 주문 대기 / 준비 포함 → 주문 준비 / 가상계좌 발급 포함 → 가상계좌 발급
- 결제완료 API는 주문 상태가 이미 확정된 시점에 호출되므로, 결제 2건의 상태가 서로 다르면 모두 주문 실패로 간주
- 웹훅은 주문 상태가 확정되지 않은 시점에서 호출될 수 있으므로, 모든 결제 상태가 동일한 경우에만 처리
- 재시도 개념이 없으므로 결제 상태가 완료가 아니고 서로 다르면 모두 실패로 간주

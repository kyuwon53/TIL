### [MEETING] 교수/선생님 회원 전환 논의
교수/선생님 회원 전환 관련 use case를 정리.
1. 교수 신청(role = prof): 직위(전임 교수/출강 강사/학원 강사/기타) → position
2. 선생님 신청(role = teacher): 직위(중고등) → position
3. 교수·선생님 모두 신청 시 교수 메뉴 + 정보교과서 메뉴 접근 가능, 직위는 위 항목을 모두 포함(전임 교수/출강 강사/학원 강사/기타/중고등) → position. 단, 이 케이스는 이번 범위에서 고려하지 않기로 함.

남은 할 일로 교수/선생님 데이터가 모두 있는 회원의 중복 제거, 운영 권한 데이터 반영, 회원 전환/수정 API, 동시 접속 어드민 계정 방지 로직 테스트가 도출됨.

### [MEETING] 로그인 아이디 중복 처리 정책
로그인 아이디 중복 시 처리 방식 논의.
- 사용 중인 아이디인 경우: "사용중인 아이디입니다" 안내
- 탈퇴한 회원의 아이디인 경우: "탈퇴한 회원입니다" 안내. 탈퇴한 아이디로 재가입이 가능한지 여부는 별도 확인 필요(통상 재가입 불가 정책이 일반적이라는 의견).

### [WORK] 이벤트 참여 API 및 본인인증 검증
클라이언트 호출 방식의 이벤트 참여 API를 설계하면서 이벤트 페이지와 직접 연결되지 않는 이벤트를 클라이언트가 안전하게 식별할 방법을 고민함. 내부 식별자를 그대로 하드코딩해 노출하는 건 위험 부담이 커서, 별도 이벤트 코드 체계가 필요하다는 결론이 나와 이번 작업은 보류(Won't Do)로 정리 (CORE-334). 별도로 브라우저 SDK로 발급받은 본인인증 요청 건이 현재 로그인한 사용자와 동일 인물인지 검증하는 API는 개발 완료 (CORE-673).

### [WORK] 결제 에러 처리 개선
PG(포트원) 결제 처리 중 SDK에서 발생하는 예외를 커스텀 에러로 감싸 응답하도록 개선. 기존엔 커스텀 에러가 원인 상세를 가려서 디버깅이 힘들었는데, 본인인증 연동 때 썼던 방식을 참고해 에러 유형별로 세분화했다 (CORE-691).

### [WORK] 주문 취소/환불 불일치 CS 대응
PG 확인 결과 결제는 성공 처리됐는데 주문 상태는 취소로 남아 환불이 안 되는 CS 문의 확인. 특정 상품 유형에 국한되지 않고 여러 건에서 같은 현상이 재현돼서 원인 분석 계속 진행 중 (OPS-651, 진행 중).

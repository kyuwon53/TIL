### [WORK] 트랙 주문/참여 상태 처리 개선
트랙 상품 주문이 취소되거나 실패하면 참여자 정보도 취소 상태로 바뀌는데, 이후 재주문할 때 유니크 제약 때문에 참여자 등록이 막히는 문제가 있었음. 취소 상태로 남아있는 경우엔 결제 대기 상태로 되돌려서 재주문 가능하게 처리 (CORE-720). 또 이미 수강 중인 상품의 다른 회차를 구매하는 경우를 위해 예약 정보를 별도로 저장해두고, 스케줄러가 회차 오픈 시점에 콘텐츠 접근 정보를 갱신하면서 안내 메일을 보내는 예약 구매 흐름 구현 (CORE-710).

### [WORK] 중복 구매/가격 응답 개선
이미 구매했거나 참여 중인 트랙 콘텐츠는 장바구니 담기·구매를 제한하는 검증 로직 추가, 마감된 트랙도 장바구니에 못 담게 처리 (CORE-724). 장바구니에서 대여 옵션으로 바꿀 때 가격 정책이 없으면 원가 대신 할인율이 0으로 잘못 적용되던 버그 수정함 (CORE-731). 이 과정에서 위시리스트/장바구니 흐름의 프론트 UX 이슈도 몇 개 확인함 (모집 마감 시 담기 버튼 노출, 옵션 담기 후 화면 즉시 반영 안 되는 문제 등) — 프론트 쪽 개선 필요 항목으로 정리해둠.

### [WORK] 본인확인 서비스 통신 방식 전환 검토
본인확인 서비스 제공사에서 보안 강화를 위해 브라우저 통신 방식에서 서버 간 통신(Server to Server) 방식으로 전환해달라는 요청이 와서 내용 확인. 전환 기한 등 요구 사항 검토했고, 추후 본인인증 개선 작업(CORE-497)과 연계해서 진행 예정 (OPS-419). 관련해서 신규 연동 테스트는 완료했고, 운영 환경 설정값 반영까지 마침. PR 반영과 개발 배포는 다음 작업으로 남김.

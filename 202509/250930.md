### [WORK] 신규 PG 도입 검토 및 PG 스위칭 설계 고려
신규 PG 관리자 문서를 검토하며 커버 가능한 작업 범위, 문제가 될 부분/질문 리스트, API 연결로 해결 가능한 부분과 신규 페이지가 필요한 부분을 파악했다.

작업순서를 리스트업해야 하며, 개발 연동을 우선적으로 진행하기 위해 테스트 연동 가이드부터 먼저 구성하기로 함. 고려해야 할 사항으로는 신규 PG와 기존 PG 사용을 손쉽게 스위칭할 수 있는지, 하나의 API로 신규/기존 PG 중 어떤 PG를 사용할지 결정할 수 있게 할 수 있는지가 있음. PG 분리/전환 이슈는 이후 레거시 결제 PG 연동 작업(HBN-2461, CORE-269)에서도 유사하게 다뤄진 주제다.

### [WORK] 사내 서브도메인 연결 작업 순서 정리
특정 사내 서브도메인 요청을 내부 API 서버로 연결하기 위한 인프라 작업 순서를 정리했다.
1. 내부 서버의 nginx 룰 수정 필요(해당 도메인 요청이 내부 API로 전달되도록).
2. 기존 로드밸런서에 리스너 룰을 추가해, 해당 도메인일 경우 내부 서버로 연결.
3. DNS에서 해당 도메인 요청이 로드밸런서로 전달되도록 레코드 추가.

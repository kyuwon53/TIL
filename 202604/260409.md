### [WORK] 플랫폼 외부 매출 관리 기능 설계 (CORE-579)
자사 플랫폼을 거치지 않고 발생하는 매출(B2B, 오프라인 세미나, 카드 결제 페이지 등)을 별도 시트 대신 운영툴에서 직접 등록·관리할 수 있도록 기능을 설계했다. 매출 유형은 고정 선택 항목과 직접 입력 항목으로 나누고, 유형별 코드 체계를 둬서 통계·필터링이 안정적으로 되도록 구조를 잡음. 외부 플랫폼 매출 중 주문 정보 추출이 가능한 건은 기존 주문 데이터로 연계하고, 불가능한 항목은 별도 매출 관리 데이터로 저장하는 방향으로 정리.

### [WORK] 코호트 상품 주문 검증 로직 설계
코호트 상품 가격 검증 방식을 고민했다. 국비 지원 상품은 가격을 정가가 아니라 자기부담금으로만 계산해야 해서, 주문 합계 계산과 가격 검증 모두 자기부담금 기준으로 맞춰야 한다는 결론에 도달. 주문서 단계에서 1차 검증을 걸어야 할지, 주문서 검증에 어떤 항목들이 있는지부터 확인이 필요하다고 정리함.
